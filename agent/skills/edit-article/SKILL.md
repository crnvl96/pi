---
name: edit-article
description: Improve written documentation or articles. Use for restructuring sections, tightening prose, preserving voice, and clarifying flow.
---

1. Divide the article into sections based on its headings. If the article has no headings, infer provisional sections.

Think about the main point of each section.

Consider that information is a directed acyclic graph: some ideas depend on earlier ideas. Make sure the order of sections and paragraphs respects those dependencies.

Confirm the sections with the user before rewriting.

2. Ask whether the user wants inline feedback, a rewritten draft, or direct file edits, unless they already specified the output mode.

3. Rewrite each section for clarity, coherence, and flow.

Preserve the author's claims and voice unless the user asks for a stronger rewrite. Do not add facts not present in the draft unless clearly marked as suggestions.

Prefer short paragraphs. Use a maximum of 240 characters per paragraph only when the user wants a terse web style.
